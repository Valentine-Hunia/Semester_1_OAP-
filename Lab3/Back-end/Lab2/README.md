## Система реєстрації на події (Варіант №7) - Лабораторна робота №3

Цей проєкт є бекендом для системи реєстрації на події. Дані зберігаються локально у вбудованій реляційній базі даних SQLite. 

Запуск серверу: npm run dev

## Ініціалізація БД:
База даних створюється автоматично при першому запуску сервера.
Файл бази даних зберігається за шляхом: ./data/app.db.
Схема (таблиці) ініціалізується автоматично (через виклик initDb()) до того, як сервер починає приймати HTTP-запити.

## Схема Бази Даних:
База даних складається з трьох пов'язаних таблиць. Увімкнено сувору підтримку зовнішніх ключів (PRAGMA foreign_keys = ON;).

**1. Таблиця Users (Користувачі)**
id (TEXT) — PRIMARY KEY (використовується UUID).

email (TEXT) — NOT NULL, UNIQUE (обмеження унікальності).

name (TEXT) — NOT NULL.

**2. Таблиця Events (Події - основна сутність)**
id (TEXT) — PRIMARY KEY (використовується UUID).

title (TEXT) — NOT NULL.

date (TEXT) — NOT NULL.

location (TEXT) — NOT NULL.

capacity (INTEGER) — NOT NULL, CHECK (capacity > 0) (місткість не може бути від'ємною або нульовою).

createdAt (TEXT) — NOT NULL (зберігається в ISO-форматі).

**3. Таблиця Registrations (Реєстрації)**
Забезпечує зв'язок 1:N (Один-до-багатьох).

id (INTEGER) — PRIMARY KEY AUTOINCREMENT.

eventId (TEXT) — FOREIGN KEY посилається на Events(id). ON DELETE CASCADE (якщо подію скасовано/видалено, всі реєстрації на неї теж видаляються).

userId (TEXT) — FOREIGN KEY посилається на Users(id). ON DELETE RESTRICT (забороняє видаляти користувача, якщо він зареєстрований хоча б на одну подію).

createdAt (TEXT) — NOT NULL.

## Приклади запитів (API)
одає новий запис у таблицю Events. База даних автоматично згенерує унікальний id (UUID) та час створення createdAt.

* **Метод:** POST
* **Ендпойнт:** /api/events
* **Headers:** Content-Type: application/json
* **Body (Тіло запиту):**

  ```json
  {
    "title": "Кібер Хакатон 2026",
    "date": "2026-10-10",
    "location": "Київ, КНУ (FIT)",
    "capacity": 150
  }
  
## Отримання списку подій (з пагінацією та пошуком)
Повертає масив подій. Підтримує фільтрацію за назвою та розбиття на сторінки (пагінацію).

* **Метод:** GET

* **Ендпойнт:** /api/events

* **Query параметри (необов'язкові):** * title — пошук за частиною назви.

        page — номер сторінки (за замовчуванням 1).

        limit — кількість записів на сторінку (за замовчуванням 5).

```json
{
    {
  "items": [
    {
      "id": "e82c4f...",
      "title": "Кібер Хакатон 2026",
      "date": "2026-10-10",
      "location": "Київ, КНУ (FIT)",
      "capacity": 150,
      "createdAt": "2026-04-10T20:46:36.230Z"
    }
  ],
  "total": 1,
  "page": 1,
  "limit": 2
}
}
